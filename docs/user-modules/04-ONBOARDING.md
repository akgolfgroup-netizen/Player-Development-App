## 6. Invitasjonssystem

### Invitasjonsflyt

```
1. Coach oppretter invitasjon
   - Genererer unik kode/lenke
   - Lagres i Invitation-tabell

2. Spiller mottar invitasjon
   - Email med lenke
   - Klikker og registrerer seg

3. Automatisk kobling
   - Spiller kobles til coach
   - Invitasjon markeres som brukt
```

### Database-modell

```prisma
model Invitation {
  id          String    @id
  coachId     String
  email       String?   // Spesifikk email eller null for Ã¥pen
  code        String    @unique
  type        String    // 'email' | 'link'
  expiresAt   DateTime
  usedAt      DateTime?
  usedBy      String?   // Player ID hvis brukt

  coach       Coach     @relation(...)
}
```

### Invitasjons-API

```
POST /api/v1/coach/invitations
  Body: { emails: string[] } | { type: 'link' }
  Response: { invitations: [...], link?: string }

GET /api/v1/invitations/:code
  Response: { valid, coachName, sportId }

POST /api/v1/invitations/:code/accept
  Body: { playerData }
  Response: { player, coach }
```
